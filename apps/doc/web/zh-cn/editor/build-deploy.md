# 构建与部署

当前版本的 Zephyr3D 项目支持将编辑器中的项目**一键打包为可部署的 Web 页面**，用于本地预览或上传到任意静态网站服务器（如 GitHub Pages、自建 Nginx 服务器等）。

---

## 一键打包为 Web 页面

### 操作步骤

1. 在编辑器菜单中，点击 **Build Project**；
2. 编辑器会自动对当前项目进行打包，生成：
   - 一个可直接在浏览器中打开的 `index.html` 页面；
   - 以及项目运行所需的相关脚本与资源文件（如 JavaScript、WASM、纹理、模型、音频等）；
3. 打包完成后，编辑器会提供一个 **Zip 压缩包** 下载链接：
   - 该压缩包内包含 `index.html` 与所有依赖资源；
   - 解压后即可在本地或服务器环境中使用。

### 部署说明（简要）

- **本地预览**：
  - 解压下载的 Zip 包；
  - 使用支持静态文件托管的本地服务器（如 `http-server`、`live-server` 等）打开目录；
  - 在浏览器中访问对应地址即可运行项目。

- **部署到线上**：
  - 将解压后的全部文件上传到任意静态网站托管环境（例如：
    - Nginx/Apache 静态目录；
    - GitHub Pages / GitLab Pages；
    - 各类对象存储静态网站托管，如 S3 + CloudFront 等）；
  - 确保 `index.html` 可被直接访问，即可在浏览器中运行最终应用。

---

## 功能待完善之处

当前构建与部署功能仍在持续迭代中，已知/规划的待完善点包括但不限于：

1. **构建配置选项较少**
   - 暂不支持在编辑器内针对构建过程进行精细化配置，例如：
     - 是否启用代码压缩 / 混淆；
     - 是否区分开发构建与发布构建（Debug/Release）；
     - 自定义输出目录结构等。
   - 未来可考虑提供“构建配置面板”，允许用户根据不同发布场景调整打包策略。

2. **环境与平台预设支持有限**
   - 当前构建流程主要面向通用 Web 环境；
   - 暂未针对特定平台（如移动端 Web、低性能设备）提供一键优化选项（例如资源压缩等级、纹理格式选择、多分辨率资源切换策略等）。

3. **构建日志与错误提示不够详细**
   - 当构建失败或部分资源打包异常时，当前错误提示可能较为简单；
   - 未来计划增强：
     - 构建日志面板；
     - 更清晰的错误类型与定位信息（例如指明是哪个脚本/资源导致构建失败）。
